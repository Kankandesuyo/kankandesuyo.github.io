# 研究依据与内容边界

本小程序不是医学、心理或身份诊断工具。它把测试定位为“自我探索材料”，原因是性取向研究通常强调多维度观察，而不是用一个分数替人做最终命名。

## 转化到本项目的设计原则

1. 多维度而非单一标签  
   题目分别覆盖情感吸引、浪漫亲密、身体吸引、过往经验、身份认同、生活选择、探索流动、低吸引与边界。

2. 吸引、经验、身份认同分开看  
   一个人的吸引对象、关系经验和自我称呼不一定完全一致，报告只呈现线索，不替用户裁定身份。

3. 给不确定和低吸引留位置  
   题库包含“尚不确定”“较少出现吸引”“需要更深安全感”等选项，避免把所有答案强行归入异性、同性或双向标签。

4. 保护隐私  
   参考敏感信息调查的基本伦理，本项目不做登录、不上传数据、不保留历史记录，也不设计排行榜或分享诱导。

5. 文案保持克制  
   题目避免露骨描写，报告避免“你就是……”式判断，使用“更接近”“可能”“当前线索”等表达。

## 题目结构

32 题分为 8 组，每组 4 题：

- 情感吸引：日常心动、欣赏、反复想起、吸引定义。
- 浪漫亲密：约会想象、长期陪伴、示好反应、恋人角色。
- 身体吸引：外貌气质、亲密接触想象、影视人物、浪漫与身体吸引关系。
- 过往经验：过去好感、关系经验、早期悸动、经历叙述。
- 身份认同：临时标签、标签舒适度、安全环境介绍、自我命名需求。
- 生活选择：作品代入、理想伴侣、社交注意、未来生活。
- 探索与流动：时间变化、难归类心动、未来可能性、确定答案需求。
- 低吸引与边界：吸引频率、亲密前提、对强烈迷恋的反应、测试期待。

## 报告边界

报告类型只用于整理答案趋势：

- 异性倾向型
- 同性倾向型
- 双向开放型
- 多性别开放型
- 低吸引探索型
- 尚在探索型

这些名称不是诊断，也不是必须采用的身份标签。用户可以接受、修改、拒绝或暂不命名。
